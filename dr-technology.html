The Loop Architecture

Most photonic chips implement matrix-vector multiplication using a spatial grid of Mach-Zehnder Modulators (MZMs), one component per matrix element, so an N-channel system needs O(N²) MZMs. Loop Labs replaces the grid with a recirculating loop: the same two columns of MZMs process successive time-bin pulses, completing the full unitary in O(N) time with a chip footprint that does not grow with matrix depth.

Spatial grid, conventional approach
N=64 requires up to 4,096 MZMs across the die. Footprint, propagation loss, and fabrication cost all scale as N².
Loop architecture · Loop Labs
N=64 needs only ~7 active MZI pairs. Matrix depth is encoded in successive time-bins (t₁, t₂, t₃…), the loop recirculates each clock cycle rather than scaling chip area.

Thin-Film Lithium Niobate Platform

Loop Labs operates on thin-film lithium niobate (TFLN), a material that offers the highest electro-optic coefficient available in any CMOS-compatible process. The Pockels effect changes the refractive index linearly with the applied field, instantaneous, noise-free, with no carrier dynamics or thermal drift.

On-Chip Optical Non-Linear Activation

The largest unsolved problem in photonic AI is nonlinear activation, applying functions like ReLU between network layers without converting light to electrons. Every silicon photonic competitor performs an optical-to-electrical-to-optical (O-E-O) conversion for every layer, consuming orders of magnitude more power than the optical compute itself.

Modularity & Scalability

A single loop chip's channel capacity is bounded by the number of supported waveguide modes. Loop Labs overcomes this ceiling through modularization in linear algebra: multiple loop-based processing units combine via block-matrix decomposition to perform matrix operations of arbitrary dimension, with the compute capacity scaling linearly with modules, not quadratically with chip area.

01 · Single Module
N-Channel Loop Unit
Each loop chip handles an N×N unitary sub-block using only N×2 MZMs, performing an O(N²) matrix in O(N) hardware
Energy per operation decreases as N grows within the module, with efficiency improving with width
Self-contained: on-chip integration of all waveguides, modulators, and detection elements
02 · Multi-Module
Block Matrix Composition
Multiple loop units tile a larger block-structured matrix in parallel via standard linear-algebra decomposition
No new physics or redesign required, modules are identical and interchangeable
Effectively unlimited matrix dimension: 100+ channels addressable with current fabrication generation
03 · System Scaling
Linear Inference Scaling
Module interconnect via on-board photonic routing or standard optical fiber
Each module operates independently, so partial deployment is immediately useful
Doubling channels doubles chips, not area², a fundamentally different scaling curve from spatial photonics or GPU die

Why Loop Labs Leads in Photonics

All photonic AI chips are not equivalent. Three structural differences, namely material platform, computational architecture, and in-loop activation, separate Loop Labs from every silicon photonic competitor.

01 · Platform
LiNbO₃ vs Silicon Photonics
r₃₃ of TFLN is 30 pm/V, while silicon has no intrinsic Pockels effect and must use carrier injection or depletion, which is slower and noisier
Silicon photonics Vπ is ~10× higher than TFLN, requiring larger drive voltages and consuming additional electrical power per switch
TFLN supports modulation bandwidths exceeding 100 GHz, while carrier-based silicon cannot approach this at comparable drive voltage
02 · Architecture
Temporal vs Spatial Compute
Spatial chips (Lightmatter, Lightelligence): N×N MZM grid, where chip area, propagation loss, and fabrication cost all grow as N²
Loop Labs: N×2 columns, with a footprint that is constant regardless of matrix depth, encoded in successive time-bins
At N=64: spatial competitors require thousands of MZMs; Loop Labs uses ~7 active MZM pairs at equivalent or higher fidelity
03 · Activation
On-Chip vs Off-Chip Nonlinearity
All major silicon photonic competitors convert optical → electrical → optical between network layers to apply nonlinear activation functions
Loop Labs implements activation via homodyne detection in the optical domain, with no O-E-O conversion and no conversion energy penalty per layer
For a 12-layer network: competitors pay 12× the O-E-O overhead per inference pass; Loop Labs pays zero

Development Roadmap

Loop Labs follows a staged deployment strategy, progressing from narrow inference tasks through convolutional workloads to full-scale language model serving, with each stage validated on demonstrated prototype results before the next begins.

Stage I · Near-term
Feedforward & MLP Inference
FNNs · MLPs · Dense layers
ZEN-1 prototype validates the core loop architecture at N=64. Initial deployment targets feedforward and multilayer perceptron inference, the highest-volume workload in recommendation, classification, and embedding models. Fully demonstrated on-chip.
Stage II · Mid-term
Convolutional & Residual Networks
CNNs · ResNets · Vision models
Extending the loop architecture to handle convolution via block-Toeplitz matrix representations. Modular multi-chip composition enables deeper residual network inference with sustained energy advantage across all layers.
Stage III · Long-term
LLMs & Edge Deployment
Transformers · LLMs · Edge AI
Full attention-mechanism and transformer inference via modular scaling. The long-term target is pulling language model serving out of hyperscale datacenters onto edge devices, from autonomous robotics to consumer electronics, where near-zero power enables on-device LLM inference for the first time.
→
Edge AI vision: The fundamental constraint preventing on-device LLM inference today is thermal power, as every watt dissipated requires battery capacity, cooling volume, and sustained power delivery. Loop Labs' near-zero static consumption and chip-scale footprint make edge LLM inference a physically tractable engineering problem rather than a thermodynamic impossibility.
